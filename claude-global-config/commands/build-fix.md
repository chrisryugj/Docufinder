# /build-fix 커맨드

## 사용법
```
/build-fix           # 현재 빌드 에러 수정
/build-fix --all     # 모든 에러 한번에 시도
```

## 동작
```
1. 빌드 실행
      │
      ▼
2. 에러 로그 수집
      │
      ▼
3. 에러 분류 (타입/의존성/설정/문법)
      │
      ▼
4. 루트 원인 식별
      │
      ▼
5. 자동 수정 적용
      │
      ▼
6. 재빌드 검증
      │
      ▼
실패시 반복 (최대 3회)
```

## 지원 에러 유형

### TypeScript
| 코드 | 설명 | 자동 수정 |
|------|------|:--------:|
| TS2307 | 모듈 못 찾음 | ✅ |
| TS2339 | 속성 없음 | ⚠️ |
| TS7006 | 암시적 any | ✅ |

### Rust
| 코드 | 설명 | 자동 수정 |
|------|------|:--------:|
| E0432 | import 오류 | ✅ |
| E0599 | 메서드 없음 | ⚠️ |

## 출력 형식
```
🔧 빌드 에러 분석 중...

발견된 에러: 3개
├── ✅ TS2307 (자동 수정됨)
├── ✅ TS7006 (자동 수정됨)
└── ⚠️ TS2339 (확인 필요)

재빌드 중...
✅ 빌드 성공!
```

## 자동 트리거
- `/verify` 실패 시 자동 호출
- 빌드 명령 실패 시 자동 호출

## 관련 커맨드
- `/verify` - 전체 검증

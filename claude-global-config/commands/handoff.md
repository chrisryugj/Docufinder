# /handoff 커맨드

## 사용법
```
/handoff              # HANDOFF.md 생성
/handoff --update     # 기존 파일 업데이트
```

## 용도
- 컨텍스트 리셋 전 현재 상태 저장
- 다음 세션으로 작업 인계
- 80k 토큰 도달 전 정리

## 생성 시점 (권장)
```
⚠️ 컨텍스트 70k 토큰 도달 시 알림
🚨 컨텍스트 85k 토큰 도달 시 강력 권장
📋 복잡한 작업 중간 체크포인트
🔚 작업 세션 종료 전
```

## 출력 형식 (HANDOFF.md)
```markdown
# HANDOFF - 2024-01-15

## 세션 요약
- 시작 시간: 14:00
- 주요 작업: 검색 기능 구현

## 완료된 작업
- [x] FTS5 인덱싱 구현
- [x] 검색 API 엔드포인트 생성
- [x] 기본 UI 컴포넌트 생성

## 진행 중인 작업
- [ ] 벡터 검색 통합 (60% 완료)
  - 현재: 임베딩 생성 완료
  - 다음: usearch 인덱스 연동

## 다음 단계
1. `src/search/vector.rs`에서 usearch 연동
2. 하이브리드 검색 병합 로직 구현
3. 검색 결과 UI 개선

## 주의사항
⚠️ 경고:
- `src/config.rs` 수정 시 재빌드 필요
- 테스트 DB 초기화 필요할 수 있음

🐛 알려진 이슈:
- 대용량 파일 인덱싱 시 메모리 사용량 높음

## 관련 파일
| 파일 | 설명 | 상태 |
|------|------|------|
| `src/search/fts.rs` | FTS 검색 | ✅ 완료 |
| `src/search/vector.rs` | 벡터 검색 | 🔄 진행중 |
| `src/api/search.rs` | 검색 API | ✅ 완료 |

## 환경 상태
- 브랜치: `feature/hybrid-search`
- 최근 커밋: `abc123` - feat: FTS5 구현
- 테스트 상태: ✅ 통과 (15/15)
- 빌드 상태: ✅ 성공
```

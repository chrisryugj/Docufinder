# /plan 커맨드

## 사용법
```
/plan [작업 설명]
```

## 예시
```
/plan 로그인 기능 구현
/plan 검색 성능 최적화
/plan 파일 업로드 기능 추가
```

## 동작
1. `planner` 에이전트 호출
2. 현재 코드베이스 자동 분석
3. 단계별 실행 계획 생성
4. 예상 위험 요소 식별
5. 사용자 승인 대기

## 출력 형식
```markdown
# 구현 계획: [기능명]

## 요구사항 요약
- ...

## 단계별 작업
1. [ ] **[작업명]** (~15분)
2. [ ] **[작업명]** (~20분)

## 수정할 파일
| 파일 | 변경 내용 |
|------|----------|

## 위험 요소
- ⚠️ ...
```

## 자동 후속 작업
- 계획 승인 → 자동 실행 모드 시작
- UI 작업 포함 → `frontend-developer` 연계
- 테스트 필요 → `tdd-guide` 연계

## 관련 커맨드
- `/verify` - 계획 실행 후 검증
- `/checkpoint` - 중간 저장점 생성

# /test-coverage 커맨드

## 사용법
```
/test-coverage               # 전체 커버리지 확인
/test-coverage [경로]        # 특정 경로만
/test-coverage --detail      # 상세 리포트
```

## 동작
```
1. 테스트 실행 (커버리지 모드)
      │
      ▼
2. 커버리지 데이터 수집
      │
      ▼
3. 분석 및 리포트 생성
      │
      ▼
4. 개선 제안
```

## 출력 형식
```
📊 테스트 커버리지 리포트

전체: 82.5%
├── Statements: 85%
├── Branches: 78%
├── Functions: 88%
└── Lines: 82%

목표 대비:
├── ✅ 일반 코드: 82% (목표: 80%)
├── ⚠️ API 모듈: 75% (목표: 90%)
└── ❌ 인증 모듈: 65% (목표: 100%)

커버리지 부족 파일:
| 파일 | 커버리지 | 미커버 라인 |
|------|----------|------------|
| auth/login.ts | 65% | 23-30, 45-52 |
| api/users.ts | 72% | 88-95 |
```

## 상세 리포트 (--detail)
```
📁 src/auth/login.ts (65%)

미커버 영역:
├── Line 23-30: 에러 핸들링 분기
│   ```typescript
│   if (error.code === 'INVALID_TOKEN') {
│     // 이 분기 테스트 없음
│   }
│   ```
│   제안: 잘못된 토큰 케이스 테스트 추가

└── Line 45-52: 세션 만료 처리
    제안: 세션 만료 시나리오 테스트 추가
```

## 커버리지 기준
| 영역 | 최소 | 권장 |
|------|------|------|
| 일반 | 80% | 90% |
| 핵심 로직 | 90% | 100% |
| 보안 | 100% | 100% |

## 관련 커맨드
- `/tdd` - 테스트 작성
- `/verify` - 전체 검증

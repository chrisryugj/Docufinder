# Anything v2.5.0 — OneDrive·네트워크 폴더 대응 + 모델 번들

**v2.4.0 의 모든 기능 + 회사망/오프라인 환경에서도 첫 실행 즉시 사용 가능**

---

## 이번 버전 핵심

### 1. 회사망/방화벽 환경 첫 실행이 즉시 가능
v2.4.0 까지는 첫 실행 시 huggingface·github 에서 ONNX Runtime + 임베딩·OCR 모델(약 14MB + 16MB)을 받아왔습니다. 사내망에서 두 사이트가 막혀 있으면 **시맨틱 검색·OCR 이 영구적으로 동작 안 했습니다.**

v2.5.0 부터는 **MSI 안에 ONNX Runtime DLL 과 PaddleOCR 모델이 함께 들어 있습니다.** 설치 직후 인터넷 없이도 OCR 이 됩니다.

> 임베딩 모델(KoSimCSE, ~106MB)은 여전히 huggingface 에서 받습니다. 회사망에서 막혀 있어도 키워드 검색·OCR·파일명 검색은 정상 동작합니다.

### 2. OneDrive 같은 클라우드 파일을 끌어내리지 않습니다
OneDrive·iCloud·Dropbox 의 "필요할 때만 다운로드(Files-On-Demand)" 폴더를 인덱싱 폴더에 추가하면, v2.4.0 까지는 인덱서가 파일을 열 때마다 OS 가 자동으로 원본을 끌어내려 **수십~수백 GB 의 트래픽과 디스크 사용**이 발생했습니다.

v2.5.0 은 클라우드 placeholder 비트를 자동 감지해 **본문 파싱은 건너뛰지만**, **파일명·크기·수정일은 정상 인덱싱**합니다. 즉, 파일명 검색은 그대로 되고, 사용자가 직접 한 번이라도 그 파일을 열어 로컬로 다운받으면 다음 인덱싱 사이클에서 본문도 들어갑니다.

### 3. 네트워크 폴더(`\\서버\공유`) 정식 지원
회사 공유 폴더 같은 UNC 경로가 v2.4.0 까지는 다음 세 가지 문제로 깨져 있었습니다:
- HWP 파서가 `\\?\UNC\…` prefix 를 잘못 잘라 한글 문서를 못 열었음
- 폴더 등록 시 표준 정규화가 서버 응답을 기다리며 수십 초 멈췄음
- 파일 변경 감시(notify) 가 SMB 위에서 동작하지 않아 새 파일을 잡지 못했음

v2.5.0 은 세 곳 모두 수정했고, 네트워크 폴더는 **30초 주기 폴링**으로 변경 감지합니다(로컬은 그대로 즉시 감지).

## 설치하기

1. 아래 **Assets** 에서 `Anything_2.5.0_x64_ko-KR.msi` 다운로드
2. MSI 파일 더블클릭 → 설치 (UAC 한 번)
3. 시작 메뉴에서 **Anything** 실행

기존 v2.4.x 사용자는 그대로 덮어쓰기 설치하면 되고, 인덱스/설정은 보존됩니다.

### 시스템 요구사항
- Windows 10 (21H2 이상) 또는 Windows 11
- RAM 8GB 이상 (16GB 권장)
- 디스크 여유 1GB 이상
- 폐쇄망·사내망에서 사용 가능 (OCR/키워드/파일명 검색은 인터넷 불필요)

WebView2, VC++ 런타임은 설치파일에 포함되어 있어 별도 설치 불필요.

### 보안 경고가 뜰 때
v2.4.0 과 동일합니다. 코드서명 인증서가 없어 다음 경고가 보일 수 있습니다.

- **Windows의 PC 보호 파란 창**: "추가 정보" 클릭 → "실행"
- **스마트 앱 컨트롤 차단 (Win11)**: MSI 파일 우클릭 → 속성 → "차단 해제"
- **Edge/Chrome 다운로드 차단**: 다운로드 창에서 "..." → "유지"
- **백신 자동 격리**: 보호 기록에서 **복원** 또는 실시간 감시 잠시 끄고 재시도

## 빠른 시작 (v2.4.0 과 동일)

폴더 등록 → 키워드 검색 → 필요하면 Gemini API 키 등록해 AI 질의응답.

| 모드 | 쓰임 |
|------|------|
| 키워드 | 기본값. 정확한 단어 매칭 (1초 이내) |
| 파일명 | Everything 스타일. 인덱싱 끝나기 전에도 즉시 사용 |
| 하이브리드 | 키워드 + 의미 검색 결합 (시맨틱 ON 필요) |
| 시맨틱 | 의미 기반 유사 문서 (시맨틱 ON 필요) |

## 새로 등록할 때 알아두면 좋은 것

### OneDrive 폴더
그대로 등록하면 됩니다. 파일명·크기·수정일은 들어가고, 본문이 들어가는 시점은 **사용자가 그 파일을 한 번이라도 열어 로컬로 내려받은 다음** 입니다.

### 네트워크 공유 폴더 (`\\파일서버\문서`)
그대로 등록하면 됩니다. 변경 감지가 30초 주기 폴링으로 동작합니다(로컬보다 다소 느림). 매핑드라이브(`Z:\…`) 도 동일하게 처리됩니다.
